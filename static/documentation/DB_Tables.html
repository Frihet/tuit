Ticket tables



ticket_issue

This is the main issue table. Contains the bulk of the data about an issue except for updates. Note that this data is not static - the status column in this table always has the latest status of the issue, not any history. We need to check the json-data to find the historic data for a specific point in time.

ticket_cidependency

This table contains issue dependencies on a configuration items (CI) from the configuration management database (CMDB). Because we want to allow the CMDB and the issue system to run on different servers without dependencies, the ci_id here is in fact not a true foreign key, it is simply an integer.


Name

Type

Description

issue

FK(ticket_issue)

Issue that has CI dependency

ci_id

Int

Id of the configuration items

description

Char(512)

Description of the CI

view_order

Int

Sorting priority, where lower number is higher priority (element with view_order = 10 comes before the component with view_order = 11)

This is single update for a given ticket. It can represent either update sent through web interface or through e-mail. Either user or contact must be set, but never both. The contact or user pointed to is the originator of this update

Description

Bool
